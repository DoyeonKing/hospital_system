# 微信开发者工具配置说明

## 问题说明

如果你遇到 `The file app.json doesn't exist` 错误，这是因为微信开发者工具指向了错误的目录。

**uni-app 项目需要先编译，然后微信开发者工具应该指向编译后的目录，而不是源代码目录。**

## 解决方案

### 方法一：正确配置项目路径（推荐）

1. **打开微信开发者工具**
2. **选择"导入项目"或"添加项目"**
3. **项目目录选择**：
   ```
   E:\hospital_system\uni_app\unpackage\dist\dev\mp-weixin
   ```
   ⚠️ **注意**：不要选择 `uni_app` 目录，必须选择 `unpackage/dist/dev/mp-weixin` 目录

4. **AppID**：`wx163a1fc0af4eee09`（已在配置中）

### 方法二：先编译再打开

1. **打开终端，进入项目目录**：
   ```bash
   cd E:\hospital_system\uni_app
   ```

2. **编译微信小程序**：
   ```bash
   npm run dev:mp-weixin
   ```
   或者
   ```bash
   npm run build:mp-weixin
   ```

3. **等待编译完成**（会显示 "Build complete" 或类似信息）

4. **打开微信开发者工具**，导入项目：
   - 项目目录：`E:\hospital_system\uni_app\unpackage\dist\dev\mp-weixin`
   - AppID：`wx163a1fc0af4eee09`

### 方法三：使用 HBuilderX（最简单）

1. **打开 HBuilderX**
2. **文件 → 打开目录** → 选择 `E:\hospital_system\uni_app`
3. **运行 → 运行到小程序模拟器 → 微信开发者工具**
4. HBuilderX 会自动编译并打开微信开发者工具

## 验证配置是否正确

打开微信开发者工具后，检查：

1. ✅ **项目目录**应该是：`E:\hospital_system\uni_app\unpackage\dist\dev\mp-weixin`
2. ✅ **文件列表**中应该能看到 `app.json` 文件
3. ✅ **控制台**不应该再出现 `The file app.json doesn't exist` 错误

## 常见错误

### 错误 1：选择了源代码目录
- ❌ 错误路径：`E:\hospital_system\uni_app`
- ✅ 正确路径：`E:\hospital_system\uni_app\unpackage\dist\dev\mp-weixin`

### 错误 2：编译目录不存在
如果 `unpackage/dist/dev/mp-weixin` 目录不存在，需要先编译：
```bash
cd E:\hospital_system\uni_app
npm run dev:mp-weixin
```

### 错误 3：编译失败
如果编译失败，检查：
1. 是否安装了依赖：`npm install`
2. Node.js 版本是否兼容（建议 14+）
3. 查看编译错误信息

## 其他错误说明

### 网络错误（可忽略）
以下错误可以忽略，不影响开发：
- `report.url.cn/sentry` - Sentry 错误上报（开发环境可忽略）
- `aegis.qq.com` - 腾讯监控（开发环境可忽略）
- `cube.weixinbridge.com` - 微信统计（开发环境可忽略）

### Browserslist 警告（可忽略）
```
Browserslist: caniuse-lite is outdated
```
这是警告信息，不影响功能。如果想修复，运行：
```bash
npx update-browserslist-db@latest
```

### 路由超时错误
如果出现 `routeTo timeout` 错误，可能是：
1. 页面加载时间过长
2. 代码有错误导致页面无法正常加载
3. 检查控制台是否有其他错误信息
